Spring Security demo

This demo highlights the stages required to build in authentication and authorisation with Spring. A class, TestRecord, was applied as a test class for authenticated CRUD operations. The key features include:

  • Building a @Configuration bean by extending WebSecurityConfigurerAdapter to implement usernames and passwords
  • Configuring controller routes and Thymeleaf templates, with and without authentication requirements
  • Building custom Login pages, with 500 (wrong credentials given) response, rememberMe and HTTPS
  • Authority entity (a list of Users with the given authority) and Roles (ADMIN, USER, etc.)
  • User entity, with a list of Authority's (a user may belong to more than one Authority)
  • Role entity, with a list of Users (maps a role name ADMIN or USER etc., to a group of users)
  • Password encryption technologies e.g. BCrypt and Cross-Site Request Forgery (CSRF) protection
  • Custom Spring annotations, with @interface, that handle routing authentication
  • Gradual refactor from WebSecurityConfigurerAdapter extension to UserDetailsService
  • Introduction of Java's Optional's (non-null objects with nullable values)
  • User login/logout timestamps and 403 (forbidden) responses and other properties (lockout, expired, modified)
