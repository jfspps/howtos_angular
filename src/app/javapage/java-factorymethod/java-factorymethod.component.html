The Factory method pattern

The Factory method pattern, as the name implies, provides a way of building multiple instances of different types without knowing precisely how the objects are created. Quite often, one builds a singleton Factory method implementation and then uses the method to build other objects as needed.

First, start with an abstract class that defines a generic message.

Then provide more specific extensions of the abstract Message class.

The abstract Factory class would instantiate the abstract Message class.

Then extend the abstract Factory method for all relevant concrete classes.

The client then requests specific instances as follows, passing a new concrete class and applying polymorphism in place of MessageCreator and Message.
