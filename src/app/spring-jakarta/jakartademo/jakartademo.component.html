Introduction to Jakarta EE for Spring developers

This demo introduces some of the basic project setup requirements to build a modern Java EE (Jakarta EE) web application resource, or WAR. The WAR can be deployed to an application server, such as Payara Micro (Payara is a derivative of Oracle GlassFish Server). The option to build a Java archive, or JAR, with Payara Micro included is available.

The key ideas and annotations used, provided in the project's readme, are:

  • Java bean and JPA configurations via XML
  • Servlets (controllers) which handle routing as extensions to the web server
  • REST endpoint exposure
  • Templates with JSP and JSF
