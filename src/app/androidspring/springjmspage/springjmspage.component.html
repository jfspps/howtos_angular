Messaging in Spring with Java Messaging Service JMS

This app demonstrates how messages can be sent between a applications, not all of which need to be Java applications. JMS is the specification (interface) setting the contract for various implementations of JMS, much like how JPA sets the specification and Hibernate sets the implementation. There are numerous implementations of JMS available, including Apache ActiveMQ and RabbitMQ for example.

One can implement messaging (which is asynchronous) as point-to-point, with sender -> queue -> consumer or with a more interconnected publish-subscribe approach. The former is demonstrated here. This project also shows:

  • Embedded ActiveMQ server (to respond to sent messages)
  • Handling of ActiveMQ server with JmsTemplate
  • Conversion between JSON (with Jackson library) and a JMS Message object
  • Send and receive listeners, along with combined send-and-receive methods
